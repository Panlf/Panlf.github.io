PostgreSQL的死锁和解锁

发表于 2018-07-20 | 分类于 数据库
字数统计: 442 | 阅读时长 ≈ 2

今天工作时候需要删除PostgreSQL库中的无用数据，因为无用数据过多，导致删除的时候出现死锁问题。

今天需要对数据库进行数据清理，就是删除PostgreSQL中的无用数据，大概50万左右的数据，使用的delete from tablename where name='..'的语句，然后开始执行，差不多2000多秒还是没有执行完(我不清楚是不是这个时候数据表被锁住了)，然后我强制关闭NAVICAT，重新执行，刚好到中午了，我放着这条SQL语句去吃午饭，然后睡了午觉起来看这条SQL执行了7000多秒还是没有执行完，然后我发觉应该是数据表被锁了，然后没办法删除数据，之后查了一下资料，释放被锁的语句，重新执行了一遍delete语句，差不多10多秒就执行完了。

解决方案

通过上述的方法顺利解决我的锁表问题。不过通过百度，还看到别人有另一种的解决方式。

  • 9.6 版本之后 pg_stat_activity视图的 waiting字段被 wait_event_type和 wait_event字段取代
  • 如果 pg_stat_activity 没有记录，可以查询 pg_locks 这个表中是否有锁定的记录。
  • pg_terminate_backend() 也可以用 pg_cancle_backend() 代替。
