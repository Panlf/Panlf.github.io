SpringCloud教程系列-Sleuth、Zipkin

发表于 2019-12-23 | 分类于 SpringCloud
字数统计: 1,686 | 阅读时长 ≈ 8

分布式链路追踪，就是将一次分布式请求还原成调用链路，进行日志记录，性能监控并将一次分布式请求的调用情况集中展示。Zipkin就是链路追踪的其中的一个实现方案，它是Twitter的一个开源项目，基于Google Dapper实现，致力于收集服务的定时数据，以解决微服务架构中的延迟问题，包括数据的收集、存储、查找和展现。Sleuth主要功能就是在分布式系统中提供追踪解决方案，并且兼容支持Zipkin。

Spring Cloud Sleuth是对Zipkin的一个封装，对于Span、Trace等信息的生成、接入HTTP Request，以及向Zipkin Server发送采集信息等全部自动完成。
